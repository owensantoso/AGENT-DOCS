# Product Concepts

Use `CONC-*` concept notes for semi-mature domain models, taxonomy, ontology, naming, or source-of-truth sketches that are too reasoned for `IDEA-*`, too non-binding for an ADR, and not yet ready to become a spec or plan.

Concepts should make confusion concrete without pretending the model is final. Promote the stable parts into specs, ADRs, plans, architecture docs, or explainers when they become authoritative.

Start from `CONC-0000-concept-title.md` or `scripts/docs-meta new concept "<title>" --domain product`.
