## How To Trace Why Something Exists

1. Start with the architecture doc's decision provenance.
2. Follow links to specs, parent plans, ADRs, or learnings.
3. For implemented work, check the relevant plan folder.
4. For session-level paper trail, check session logs.
5. For commit-level archaeology, look for commit trailers.

## What Not To Trust Alone

- Do not treat chat history as the source of truth.
- Do not implement from an implementation brief without reading the parent plan.
- Do not treat architecture docs as proof that something is already built.
- Do not treat unchecked historical checkboxes as definitive status without checking current state and git history.
